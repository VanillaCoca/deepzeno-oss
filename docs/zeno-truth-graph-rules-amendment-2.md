## 0. 本修订案改变什么、不改变什么

**改变**:v1 §1.1「全貌层默认不画任何关系线」由铁律降为**默认值 + 有范围例外**——一类经过严格限定的「安静依赖线」在全貌层常显。

**不改变**:两层视图、同屏并置、语义分层五条带(修订案 №1)、颜色唯一语义、truth 只读、色盲冗余、其余全部条款。修订案 №1「位置承担结构」依然是主结构手段;安静线是它的**补充**,不是替代。

## 1. 为什么修订(诊断)

修订案 №1 用五条带让「生命周期」可读,但**依赖关系**仍然只有点开(选中→单链卡)才可见。产品所有者裁定:前置假设依赖与事件先后关系是判断状态的一等信息,「哪个决定建立在哪个假设上」必须**不点开也能看出来**——这正是 §6.1「让用户清楚知道判断建立在哪些假设之上」的全貌层落实。

Obsidian hairball 教训(不画全量线)依然成立。本案的答案不是画全部线,而是**只画承重的那几条**。

## 2. 安静依赖线:范围(硬性)

一条边要在全貌层常显,必须**同时**满足:

1. 关系类型 ∈ { `depends_on`, `implies` } —— 前置依赖与推导先后。`resolves`/`refines` 继续用 IBIS 缩进表达(候选挂问题下),`supersedes`/`contradicts` 永不画线(用 rejection 节点与删除线表达)。
2. 两端都是**同一 topic** 的顶层节点(无 `parentId`)。跨 topic 关系不画(topic 间靠留白分隔的原则不破)。
3. 两端行**当前可见**(不在收起的折叠区内)。折叠导致端点不可见时,依赖信息由被依赖方节点上的「⚓N」角标兜底(见 §4)。

数量论证:单 topic 顶层节点通常 <15,满足以上条件的边通常 <10 条,且走左侧 gutter 正交通道(§3)——不构成 hairball。

## 3. 视觉与交互(全部走 token)

- **路由**:v1 §4.4 正交通道,通道位于泳道左侧 gutter(`--z-lane-gutter`,通道间距 `--z-lane-channel-gap`),跨距短走内侧;拐角小圆角(`--z-corner-radius`);箭头(`--z-arrow`)水平指入子节点行左缘。
- **层次**:常态线用 `--z-line`(fg 22%,退到背景)——「安静」的实现;hover 节点或 hover 线本身 → 该边升到 `--z-arrow` 前景并浮出标签;选中节点 → 链上边高亮,无关边降至 `--z-focus-edge-faint`。全部过渡 `--z-transition`,一次性入场微动画(实线描画 / 虚线淡入),无持续动效。
- **线型语义**(沿用 v1 §4.3):父端是 `hypothesis`(可证伪假设,天然未定)→ **虚线**;父端是 constraint/decision 等确定前提 → 实线。
- **标签**:悬停或链上高亮时显示边的自由文本 `label`(无则回退关系词 `graph.rel*`),`--z-edge-label` / `--z-font-edge`,底色 `--z-edge-label-bg`。常态不显示——标签是「必要时写清」,不是常驻噪音。
- **汇聚序号常显**(v1 §5.3 的加强):一个节点有 ≥2 条常显入边时,每条入边入口处的 ①② 徽记**常态可见**(`--z-font-badge`,`--z-text-3` 调)——「一个节点指向两个前置假设」正是本案要不点开就能读出的核心场景。

## 4. 前提带可见性(对修订案 №1 §2 的细化)

- 前提折叠条默认**展开**当 topic 前提数 ≤ 4(展开后前提行是安静线的合法端点);> 4 时保持折叠(v1 §6.3 的密度纪律)。
- 折叠状态下,依赖这些前提的节点行尾显示「⚓N」角标(N = 该节点依赖的前提数)——端点看不见时,依赖数量仍可见。展开后角标让位于真实连线。
- 用户手动开合折叠条即时生效并同步给 overlay 重测。

## 5. 与既有规则的关系裁决

| 条款 | 裁决 |
|---|---|
| v1 §1.1 全貌不画线 | 改为默认值;§2 范围内的安静线是唯一例外 |
| v1 §2 选中只点亮该链 | 不变;安静线在选中态服从三级透明度(链上高亮/无关降淡) |
| 修订案 №1 位置承担结构 | 不变;安静线补充「带间依赖」这一位置表达不了的信息 |
| v1 §4 视觉语言 | 完全沿用(线型/箭头/标签 token 无一新增语义) |
| v1 §7 truth 只读 | 不变;线只读,无任何编辑交互 |

## 附:本修订案新增决策清单

1. 安静依赖线:depends_on/implies、同 topic、顶层、可见端点,四条件缺一不画
2. 常态背景化(--z-line),hover/选中升前景;标签按需浮现
3. hypothesis 前提入边用虚线(未定语义),其余实线
4. 汇聚 ①② 常显
5. 前提折叠条 ≤4 默认展开;折叠时 ⚓N 角标兜底
6. 入场一次性微动画(描画/淡入),永不循环
